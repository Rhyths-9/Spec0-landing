Stop pasting OpenAPI specs into Cursor. Stop guessing if they're current.

Two ways to give an agent your Orders API. One feeds it a wall of stale YAML and hopes. The other answers the question it actually asked.

Pasted at 9am. By lunch the spec changed. The agent doesn't know.

~400 tokens

HOW IT WORKS

Three steps. Then your agents just work.

01
Push your spec

One CLI install, OAuth login, and your spec lives in spec0's registry — scoped to your org, versioned, linted.

$ npm install -g @spec0/cli
$ spec0 auth login
$ spec0 push openapi.yaml
✓ Published → orders-api v2.1
02
Wire your MCP client

One command writes the MCP config. No JSON editing. Cursor, Claude Code, or any MCP client immediately knows your API exists.

$ spec0 mcp install --client cursor
# or: --client claude-code
✓ Config written to ~/.cursor/mcp.json
✓ 12 APIs now queryable
03
Ask your agent anything

The agent retrieves only the operation it needs — always current, ~400 tokens, never a full spec dump. You get correct code on the first try.

FAQ

Before you deploy.

Where does my OpenAPI spec live? +
Encrypted at rest in EU-hosted Postgres. You can delete any spec via CLI or API at any time. We don't share, train on, or expose your specs.
Does spec0 see my LLM prompts? +
No. Your agent runs on your OpenAI or Anthropic account. spec0 only serves spec content over MCP — we never see your prompts, your model output, or your API key.
How is this different from Postman or SwaggerHub? +
They're API workspaces built for humans clicking through docs. spec0 is built for agents — a registry whose primary consumer is an MCP query from Cursor or Claude Code. We do the registry table stakes (lint, diff, mocks, ownership) but the product is the live MCP layer on top, not the GUI.
What if the MCP server goes down? +
Public uptime history at status.spec0.io. If MCP is unreachable, your agent falls back to whatever context it already has — your code doesn't break.
Can I self-host? +
The mock server is open source today (MIT). The CLI is open source (Apache-2.0). Self-host of the full registry + MCP layer is on the roadmap for Enterprise.
Why pay? +
You could build it. A webhook for spec drift, a CI gate, a Slack notifier, an OAuth-protected MCP server, a registry. We did. Roadmap covers another year.
