FAQ

Reasonable questions
before you push.

Where does my OpenAPI spec live? +
Encrypted at rest in EU-hosted Postgres. You can delete any spec via CLI or API at any time. We don't share, train on, or expose your specs.
Does spec0 see my LLM prompts? +
No. Your agent runs on your OpenAI or Anthropic account. spec0 only serves spec content over MCP — we never see your prompts, your model output, or your model key.
How is this different from Postman or SwaggerHub? +
They're API workspaces built for humans clicking through docs and collections. spec0 is built for agents: a registry whose primary consumer is an MCP query from Cursor or Claude Code. We do the registry table stakes — lint, diff, mocks, ownership — but the product is the live MCP layer on top, not the GUI.
What if the MCP server goes down? +
Status page at status.spec0.io with public uptime history. If MCP is unreachable, your agent falls back to whatever context it already has — your code doesn't break.
Can I self-host? +
The mock server is open source today (MIT). The CLI is open source (Apache-2.0). Self-host of the registry + MCP layer is on the roadmap for Enterprise customers.
Why pay? +
You could build it yourself. A webhook for spec drift, a CI gate, a Slack notifier, an OAuth-protected MCP server, a registry. We did. Roadmap covers another year.
